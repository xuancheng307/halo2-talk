PLONKish 閘的例子集 — 同一張表、同一組欄，多種閘並存（＋複製 / 查表對照）

← 回報告
重點先講：PLONKish 只有一張表、一組固定的欄。所有閘共用同一組 advice 欄（a,b,c…），每種閘配一個自己的 selector 欄（q+, q×, q⁵…）。某列哪個 selector=1，就在那列驗哪條閘；沒被選到的閘，selector=0 把它歸零、不生效。所以欄位不是每個閘自己一套，是大家共用。
① degree＝最高項幾個變數相乘（≤~6）。 ② rotation 引用相鄰列（近）；複製約束把任意兩格綁相等（遠）。

先搞懂：閘不是「某幾列」，是「蓋在每一列的一條規則」

你可能以為一個閘＝表格裡某幾列。其實相反：閘是一條方程式，它往下蓋在每一列，每列各代入自己那列的數字、都必須 = 0。sel 只決定哪幾列這條規則生效。
那「填數字」怎麼變成「檢查約束」？
prover 在外面算好、把結果填進格子（這就是 witness）。閘不負責算、負責「檢查」 —— 它要求你填的數字讓那條方程式每列 = 0；填錯一個 → 某列 ≠ 0 → 整除壞掉、證不出來。把「每列 = 0」變成一條多項式整除（Halo2 的關鍵一步）： \[ \underbrace{g(\omega^i)=0\ \ \forall i}_{\text{每列都滿足}}\iff \underbrace{Z_H \mid g}_{Z_H(X)=X^{n}-1}\iff \exists\,t(X):\ g(X)=t(X)\,Z_H(X) \] 驗證者不重算你的計算，只在一個隨機點 \(\zeta\) 檢查 \(g(\zeta)=t(\zeta)\,Z_H(\zeta)\) 就信 —— 一個點，換來對整條多項式的信任。
同一格、三種鏡頭（回答你的洞察）：同一個格子（例如 x 在第 1 列）可以同時被閘用（出現在方程式裡）、被複製約束綁（跟別列的 x 相等）、被查表範圍檢查（值 ∈ 表）。三種約束不是各佔不同列，是對同一批數字的三種不同檢查 —— 閘看「本列＋鄰列」、複製看「任意兩格相等」、查表看「值在不在表裡」。

每條閘長什麼樣（都跑在上面那張表）

① 加法 deg 1
\(q_{+}\cdot(a+b-c)=0\)

純加法、最低次。上表第 0 列。

② 乘法 deg 2
\(q_{\times}\cdot(a\cdot b-c)=0\)

兩格相乘 —— R1CS 的上限就停在這。第 1 列。

③ 高次 deg 5
\(q_{5}\cdot(c-a^{5})=0\)

一條約束做五次方（Poseidon S-box，PF2 用）；R1CS 要拆 3 條乘。第 2 列。

④ 布林 / 選擇 deg 2
\(q_{?}\cdot a(a-1)=0\)

逼一格成 0/1；再配 \(out=a\,x+(1-a)y\) 就是 2-路 MUX（分支）。第 3 列。

⑤ 跨列 rotation
\(q_{\Sigma}\cdot(a+a_{\downarrow}+a_{\downarrow\downarrow}-b)=0\)

次數只有 1，特殊在引用相鄰列（同一欄 a、往下兩列）。第 4 列。

⑥ 我們的 Horner PF1 核心
\(q_{H}\cdot(\mathrm{acc}\cdot x+c-\mathrm{acc}_{\downarrow})=0\)

乘加 + 引用下一列，一列一步算多項式。論文再加 ÷S 截斷 + byte 查表＝完整定點 kernel。

注意：上面六條都是「閘」（多項式＝0，每列檢查；⑤⑥ 靠 rotation 讀鄰列）。但複製約束和查表根本不是閘 —— 它們是另外兩套平行的約束，看下面。

三種約束機制 —— 只有第 ① 種叫「閘」

很多人以為所有約束都寫進 gate。其實 PLONKish 有三種平行的約束，宣告方式、能跨幾列、結構存哪、何時被檢查都不同。
機制怎麼宣告（halo2 API）能跨幾列固定結構存哪每次證明額外產生
① 自訂閘 gatemeta.create_gate：寫一條多項式 = 0本列＋固定偏移鄰列（rotation，如 \(a(\omega X)\)）selector / fixed 承諾 → vk—（閘本身不另造多項式）
② 複製約束 permutationenable_equality + region.constrain_equal任意兩格（相隔多遠都行）置換 \(\sigma\) 多項式承諾 → vk / pkgrand-product Z_P（每證明承諾一次）
③ 查表 lookupmeta.lookup：（輸入式 → 表欄）逐列把值丟進表表（fixed 欄）承諾 → vkgrand-product Z（每證明承諾一次）
時間線（回答「什麼時候做的約束」）：configure 把三種全宣告 → keygen 把固定結構（selector / \(\sigma\) / 表）烤進 vk·pk → prove prover 臨時造出 grand-product \(Z_P\) / \(Z\) 並承諾（這幾條不在 pk，每次證明才生）→ verify 最終 \(G(\zeta)=t(\zeta)\,Z_H(\zeta)\) 裡，閘項＋置換項＋查表項全用 \(\alpha\) 的次方折成一條 \(G\)，一次驗完 —— 這才是三種約束真正「被檢查」的一刻。

② 複製 與 ③ 查表 長什麼樣（不是閘，補上視覺）

② 複製約束（permutation）任意兩格 = 相等

把相隔很遠的任意兩格綁成相等（這裡 a 欄第 0 列 ＝ c 欄第 3 列）。rotation 搆不到的，靠它。例：demo 每列的 x 都綁成同一個 3。

③ 查表（lookup）值 ∈ 表

不是等式，是「值必須落在預先定義的表裡」。範圍檢查（拆 byte 查 0–255）、SHA/Keccak、定點截斷餘數都靠它。

複製的數學 —— 排列大乘積怎麼把「兩格相等」變成多項式

「這兩格要相等」也不像方程式。複製約束用一個排列 σ ＋ 一條連乘積，把它變成「兩個多重集合相等」的隨機點檢查。
① 核心構造（grand product）：把每個格子編碼成一對 (值, 位置)。用挑戰 \(\beta,\gamma\) 把一對壓成一個指紋 \(v+\beta\cdot\text{位置}+\gamma\)，逐格累乘成一條跑動積 \(Z\)： \[ Z_{i+1}=Z_i\cdot\frac{v_i+\beta\,\mathrm{id}(i)+\gamma}{\,v_i+\beta\,\sigma(i)+\gamma\,},\qquad Z_0=Z_n=1 \] 分子把每個值配它自己的位置 \(\mathrm{id}(i)\)、分母配 σ 排列後的位置 \(\sigma(i)\)；要求 \(Z\) 繞一圈回到 1。
② 為什麼這保證相等：\(Z_n=1 \iff \prod_i\!\big(v_i+\beta\,\mathrm{id}(i)+\gamma\big)=\prod_i\!\big(v_i+\beta\,\sigma(i)+\gamma\big) \iff\) 多重集合 \(\{(v_i,\mathrm{id}(i))\}=\{(v_i,\sigma(i))\}\)（\(\beta,\gamma\) 隨機 → 兩堆指紋乘積相等，幾乎必代表同一個多重集合）。位置被 σ 重排了、兩堆卻還相等 —— 唯一可能就是 σ 串在一起的那些格子，值都一樣。
③ 例子 + 電路化：demo 每列的 x 都要 = 3 → σ 把這些 x 格的位置串成一個環（0→1→2→0）。連乘 \(Z\) 從 1 出發、繞完環又回到 1，就證明了三個 x 相等。\(Z\) 是承諾的欄；邊界 \(\ell_0(X)\cdot(Z(X)-1)=0\)（起點 = 1）、每列的乘積規則、加 ZK 的盲化列，全是低次閘 → 折進 \(G=t\cdot Z_H\) → 隨機點 \(\zeta\) 一次驗完。σ 的承諾在 keygen 烤進 vk/pk；\(Z\) 每次證明才生（對照前面那張「三種約束」表）。
出處：排列論證來自 PLONK（Gabizon–Williamson–Ciobotaru, ePrint 2019/953，§5 permutation argument）；halo2 的具體建構見 The halo2 Book — Permutation argument（用 \(\delta_i\cdot\omega^j\) 給每一欄不同陪集當「位置標籤」，是 Vitalik 對 PLONK 的優化）。

查表的數學 —— logUp 怎麼把「值 ∈ 表」變成多項式

「值要落在表裡」聽起來不像方程式。logUp 用一條漂亮的恆等式，把它變成可在隨機點抽查的多項式約束。
① 核心恆等式（對數導數 / log-derivative）： \[ \sum_{i=0}^{n-1}\frac{1}{X+f_i}\;=\;\sum_{j=0}^{m-1}\frac{m_j}{X+t_j} \] 其中 \(f_i\)＝被查的值、\(t_j\)＝表的值、\(m_j\)＝\(t_j\) 被查到的次數（重數）。這條有理函數等式成立 \(\iff\) 多重集合 \(\{f_i\}\subseteq\{t_j\}\)（即每個查值都在表裡）。
為什麼？看「極點」：左邊在 \(X=-f_i\) 各有一個極點、右邊在 \(X=-t_j\) 有重數 \(m_j\) 的極點；兩個有理函數相等 \(\iff\) 極點位置與重數完全對上 \(\iff\) 每個 \(f_i\) 都剛好等於某個 \(t_j\)。原文一句講透：「logarithmic derivatives translate products of linear factors into sums of their reciprocals, turning zeroes into simple poles of same multiplicity.」
最容易卡的點：「表 T 比被查的 F 多很多，兩邊怎麼會相等？」 —— 關鍵在 \(m_j\)：沒被查到的表值，\(m_j=0\)，那一整項變 0、消失。大表根本不會整個加進來。把表縮成 \(\{0,1,2,3,4,5\}\) 看清楚，假設你查了 3 次：值 3、3、0（3 查兩次、0 查一次）：
· 左邊（每個查值一項）：\(\dfrac{1}{X+3}+\dfrac{1}{X+3}+\dfrac{1}{X+0}=\dfrac{2}{X+3}+\dfrac{1}{X+0}\)
· 重數：\(m_0=1,\ m_3=2\)，其餘 \(m_1=m_2=m_4=m_5=0\)
· 右邊：\(\dfrac{1}{X+0}+\dfrac{2}{X+3}+\big(\tfrac{0}{X+1}+\tfrac{0}{X+2}+\tfrac{0}{X+4}+\tfrac{0}{X+5}\big)\) —— 後面那串全是 0、消失
→ 兩邊都剩 \(\dfrac{1}{X+0}+\dfrac{2}{X+3}\)，當然相等。表多大都沒差，沒查到的項自動歸零；兩邊最後都化簡成「每個用到的值：被查幾次 ∕ (X＋值)」。
那它怎麼抓出「不在表裡的值」？假設你偷查一個表裡沒有的 9：左邊多一項 \(\dfrac{1}{X+9}\) → 在 \(X=-9\) 有個極點；但右邊只有表值 \(\{0..5\}\) 的極點，湊不出 \(X=-9\)（9 不是任何 \(t_j\)，沒有 \(m_j\) 能掛上去）→ 兩邊不相等 → 作弊被抓。所以「等式成立 ⟺ 每個查值都在表裡」。
那 \(\frac{1}{X+f}\) 到底怎麼「壓成多項式」去 KZG 承諾？（延續上面同一個例子）
① 先破關鍵點：\(\frac{1}{X+f}\) 不是多項式、不能直接承諾。真正進電路時，第一步把 X 固定成一個隨機數 \(\beta\)（Fiat–Shamir 抽的）—— 一旦 \(X=\beta\)，\(\frac{1}{\beta+f}\) 就只是一個具體數字，prover 在外面算得出來。\(X\) 只是「看極點、懂原理」用的；承諾的不是 \(\frac{1}{X+f}\)，是「一整欄算好的數字」插值成的多項式。
② 把例子排成 trace 的欄（\(\beta\) 已抽好；\(u,w\) 欄填的全是 prover 算好的具體數字，就像 Horner 例子裡 acc 欄填 2,9,31,98）：

③ 每欄插值成一條多項式 → KZG 各承諾：\(f,\ u,\ t,\ m,\ w\) 再加一條累加欄 \(S\)，每欄（一串數字）插值成 \(f(Y),u(Y),\dots\)（\(Y\)＝子群 H 上的位置變數），用 KZG 各壓成一個群元素 —— 這就是收尾全景圖 lookup 框裡那幾條。
④ logUp 規則 → H 上的多項式恆等式（折進 G）：
· 逆元對：\(u(Y)\cdot(\beta+f(Y))-1=0\) —— 強制 \(u\) 真的是 \(\tfrac{1}{\beta+f}\)（度數 2 的閘）
· 表側：\(w(Y)\cdot(\beta+t(Y))-m(Y)=0\) —— 強制 \(w=\tfrac{m}{\beta+t}\)
· 累加：\(S(\omega Y)-S(Y)-\big(u(Y)-w(Y)\big)=0\)（逐列累加 \(\textstyle\sum u-\sum w\)）
· 邊界：\(S\) 從 0 出發、繞完回到 0（因 \(\sum u=\sum w\)）
全是低次多項式 → 折進大 \(G\) → 除以 \(Z_H\) 得商 → 在隨機點 \(\zeta\) 一次驗完。
切記兩個不同的變數：\(X\)（看極點用，固定成隨機 \(\beta\)）≠ \(Y\)（插值位置，跑遍 H）。承諾的是 \(u(Y)\) 這種「欄多項式」，不是 \(\frac{1}{X+f}\)。
② 怎麼變成電路約束（三步）：
· 抽隨機 \(\beta\)（Fiat–Shamir），把 \(X\to\beta\)：整條有理等式縮成一個體元素等式（Schwartz–Zippel：隨機點成立就幾乎處處成立）。
· 除法用「逆元見證」拆掉：電路不能寫 \(1/(\beta+f)\)，改讓 prover 填 \(u=1/(\beta+f)\)，閘只檢查 \(u\cdot(\beta+f)=1\)（degree-2）。表側同理配重數 \(m_j\)。
· 累加 = 0：用一條跑動加總欄逐列累加 \(\sum u_f-\sum m\,u_t\)，邊界強制總和 \(=0\)。
三者全是低次閘＋邊界約束 → 折進 \(G=t\cdot Z_H\) → 在隨機點 \(\zeta\) 一次驗完。被查值、重數、逆元都是 prover 填的見證，閘只負責檢查。
出處（別搞混）：deck 與本頁用的這條公式是 logUp —— Haböck,《Multivariate lookups based on logarithmic derivatives》, ePrint 2022/1530。更早的 plookup（Gabizon–Williamson, ePrint 2020/315）目標相同，但作法是「把被查欄與表排序串接後用 grand-product 比相鄰差」，式子較繁；原版 zcash halo2 走 plookup 路線，PSE 版改用 logUp（mv-lookup）。

收尾全景：三種約束 → 折成 G → KZG 承諾 → 在 ζ 驗一條

把這幾天問通的串成一張圖：算術化層（左）產生的多項式，全進承諾層 KZG（右），最後濃縮成「一個點 ζ 上的一條等式」。
