重點先講：PLONKish 只有一張表、一組固定的欄。所有閘共用同一組 advice 欄（a,b,c…），每種閘配一個自己的 selector 欄（q+, q×, q⁵…）。某列哪個 selector=1，就在那列驗哪條閘；沒被選到的閘，selector=0 把它歸零、不生效。所以欄位不是每個閘自己一套，是大家共用。
① degree＝最高項幾個變數相乘（≤~6）。 ② rotation 引用相鄰列（近）；複製約束把任意兩格綁相等（遠）。

一張通用表，多種閘靠 selector 切換

同一組欄 a b c（advice）＋ 每種閘一個 q*（selector）。每列只有一個 q*=1（綠色），那條閘就在該列生效、讀同一組 a b c；·＝該列用不到的格子。

看懂三件事：(1) 欄位固定就這幾個，每條閘都讀同一組 a b c；(2) 每列只有一個 q*=1，決定那列驗哪條閘；(3) 第 4 列的 qΣ 只在那列生效，卻靠 rotation 往下讀黃框那三格 a（第 4/5/6 列＝2,3,4）—— 方程式裡 a(X)=本列、a(ωX)=下列、a(ω²X)=下下列。不是多了欄，是同一條式子用 ω 指向下面的列。

每條閘長什麼樣（都跑在上面那張表）

① 加法 deg 1
\(q_{+}\cdot(a+b-c)=0\)

純加法、最低次。上表第 0 列。

② 乘法 deg 2
\(q_{\times}\cdot(a\cdot b-c)=0\)

兩格相乘 —— R1CS 的上限就停在這。第 1 列。

③ 高次 deg 5
\(q_{5}\cdot(c-a^{5})=0\)

一條約束做五次方（Poseidon S-box，PF2 用）；R1CS 要拆 3 條乘。第 2 列。

④ 布林 / 選擇 deg 2
\(q_{?}\cdot a(a-1)=0\)

逼一格成 0/1；再配 \(out=a\,x+(1-a)y\) 就是 2-路 MUX（分支）。第 3 列。

⑤ 跨列 rotation
\(q_{\Sigma}\cdot(a+a_{\downarrow}+a_{\downarrow\downarrow}-b)=0\)

次數只有 1，特殊在引用相鄰列（同一欄 a、往下兩列）。第 4 列。

⑥ 我們的 Horner PF1 核心
\(q_{H}\cdot(\mathrm{acc}\cdot x+c-\mathrm{acc}_{\downarrow})=0\)

乘加 + 引用下一列，一列一步算多項式。論文再加 ÷S 截斷 + byte 查表＝完整定點 kernel。

注意：上面六條都是「閘」（多項式＝0，每列檢查；⑤⑥ 靠 rotation 讀鄰列）。但複製約束和查表根本不是閘 —— 它們是另外兩套平行的約束，看下面。

三種約束機制 —— 只有第 ① 種叫「閘」

很多人以為所有約束都寫進 gate。其實 PLONKish 有三種平行的約束，宣告方式、能跨幾列、結構存哪、何時被檢查都不同。
機制怎麼宣告（halo2 API）能跨幾列固定結構存哪每次證明額外產生
① 自訂閘 gatemeta.create_gate：寫一條多項式 = 0本列＋固定偏移鄰列（rotation，如 \(a(\omega X)\)）selector / fixed 承諾 → vk—（閘本身不另造多項式）
② 複製約束 permutationenable_equality + region.constrain_equal任意兩格（相隔多遠都行）置換 \(\sigma\) 多項式承諾 → vk / pkgrand-product Z_P（每證明承諾一次）
③ 查表 lookupmeta.lookup：（輸入式 → 表欄）逐列把值丟進表表（fixed 欄）承諾 → vkgrand-product Z（每證明承諾一次）
時間線（回答「什麼時候做的約束」）：configure 把三種全宣告 → keygen 把固定結構（selector / \(\sigma\) / 表）烤進 vk·pk → prove prover 臨時造出 grand-product \(Z_P\) / \(Z\) 並承諾（這幾條不在 pk，每次證明才生）→ verify 最終 \(G(\zeta)=t(\zeta)\,Z_H(\zeta)\) 裡，閘項＋置換項＋查表項全用 \(\alpha\) 的次方折成一條 \(G\)，一次驗完 —— 這才是三種約束真正「被檢查」的一刻。

② 複製 與 ③ 查表 長什麼樣（不是閘，補上視覺）

② 複製約束（permutation）任意兩格 = 相等

把相隔很遠的任意兩格綁成相等（這裡 a 欄第 0 列 ＝ c 欄第 3 列）。rotation 搆不到的，靠它。例：demo 每列的 x 都綁成同一個 3。

③ 查表（lookup）值 ∈ 表

不是等式，是「值必須落在預先定義的表裡」。範圍檢查（拆 byte 查 0–255）、SHA/Keccak、定點截斷餘數都靠它。

一句話：一張 2ᵏ 列的表、一組固定的欄；每種閘配一個 selector 欄，共用 advice 欄、靠 selector 決定每列驗哪條；rotation 讀鄰列、複製約束接任意兩格。 · ← 回報告
