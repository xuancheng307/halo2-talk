PART I · 在做什麼

Halo2 的家譜

原文 · Halo “We obtain and implement Halo, the first practical example of recursive proof composition without a trusted setup, using the discrete log assumption over normal cycles of elliptic curves.” — 首個「免信任設定」的實用遞迴證明組合，建立在橢圓曲線離散對數假設上。 Bowe, Grigg, Hopwood — IACR ePrint 2019/1021 ↗

一句話定位

Halo2 不是「一個」協議，而是 「算術化層 + 多項式承諾層」 的組合框架 — 你描述電路，它負責產生／驗證證明。

PART I · 那個分叉的核心

可信設定（Trusted Setup）：在做什麼 · 跟 IPA 差在哪

前面反覆提到「KZG 需要可信設定、原版 IPA 不用」。它到底在做什麼？為什麼有的需要、有的不用？

可信設定在做什麼
一次性「儀式」產生公共參數 SRS：從一個隨機秘密 \(\tau\) 算出 \(\{[\tau^i]_1\}\dots\)（承諾要用）。關鍵是 \(\tau\) 是 toxic waste（有毒廢料） —— 算完必須銷毀；任何人若留著 \(\tau\) 就能偽造假證明。降低風險：用多方計算儀式（powers-of-τ），只要一個參與者誠實銷毀自己那份，整體就安全；通用可更新版還能讓新人持續加入增強。
方案要 setup？為什麼代價 / 好處
原版 Halo2 · IPA不要Pedersen/IPA 用公開的隨機基（hash 出來、nothing-up-my-sleeve），沒把秘密藏進參數 → 沒有 toxic waste無設定風險、遞迴友善；但驗證 線性、proof 較大
Groth16每電路一次CRS 綁特定電路換電路要重辦儀式，最不通用
Halo2 PSE · KZG（我們）一次 · 通用 · 可更新承諾/開啟靠藏了 \(\tau\) 的結構化 SRS（配對檢查要 \([\tau]_2\)）一次性可信設定；但所有電路共用、可更新降風險，換得通用可更新 + 上鏈便宜
差異的本質：承諾方案有沒有把秘密藏進公共參數。藏了（KZG 的 \(\tau\)）→ 就需要可信設定把它安全產生並銷毀；沒藏（IPA 用公開隨機基）→ 根本不需要。我們選 KZG，是用「一次性、通用可更新的 setup（不必每電路重辦）」換「BN254 上鏈驗證便宜、承諾固定大小」—— 注意：證明小不是它的賣點（Groth16 反而更小），通用設定 + 自訂閘/查表才是。τ 的形式、SRS 細節見後面 KZG；pk/vk 怎麼從 SRS 生出見〈Setup → Keygen〉。
PART I · 現代範式

這套現代範式的名字叫 PLONK

從 Sigma / Schnorr、Groth16 到現代通用 SNARK 之間的範式轉移，就是 PLONK（2019） —— 而 Halo2 是它的一個實作。

原文 · PLONK “We present a universal SNARK construction with fully succinct verification, and significantly lower prover running time … we focus on ‘Evaluations on a subgroup rather than coefficients of monomials’, which enables simplifying both the permutation argument and the arithmetization step.” Gabizon, Williamson, Ciobotaru — ePrint 2019/953 ↗

PLONK 帶來的三個「新」

  • 通用 + 可更新 SRS：一次設定，任意電路共用（不像 Groth16 每電路一次）。
  • 多項式 IOP：電路 → 多項式約束，用承諾 + 隨機點檢查取代「reduce 到單一困難問題」。
  • 自訂閘 + 查表（UltraPLONK / Plonkish）：表達複雜運算。

Halo2 = PLONKish 算術化 + 多項式承諾（原版用 IPA、我們的 PSE 版用 KZG）。你今天要懂的「新東西」就是這個 PLONK 家族。

原文 · Relation R “… we write down a relation, R, that specifies which combinations of public and private inputs are valid.”
原文 · Circuit “… we should distinguish between the relation R, and its implementation to be used in a proof system. We call the latter a circuit.”
原文 · Witness “The private inputs and advice values are collectively called a witness.” The halo2 Book — Proof systems ↗

PART II · 第 2 層

自訂閘：每條規則 = 一個 恆等於 0 的多項式

🎯 真正要驗的就這條：\(s\cdot(\mathrm{acc}\cdot x+c-\mathrm{acc}')=0\) —— 被選到（\(s{=}1\)）的每一列都得滿足這條規則。下面的圖與例子都在解釋這條怎麼讀、怎麼套。
原文 · Polynomial constraints “These are multivariate polynomials over F that must evaluate to zero for each row. The variables … may refer to a cell in a given column of the current row, or a given column of another row relative to this one (with wrap-around …).”
原文 · Gate “… a set of constraints controlled by a set of selector columns that are designed to be used together, [we refer to] as a gate.” The halo2 Book ↗
這條閘的多項式形式（demo）
\[ s\cdot\big(\mathrm{acc}\cdot x + c - \mathrm{acc}'\big)=0,\qquad \mathrm{acc}'=\mathrm{acc}\ @\ \texttt{Rotation::next} \]
套我們的例子（第 0 列）：\(s{=}1,\ \mathrm{acc}{=}2,\ x{=}3,\ c{=}3,\ \mathrm{acc}'{=}9\ \Rightarrow\ 1\cdot(2\cdot3+3-9)=0\ ✓\)（一路逼到第 3 列就是 98）。
關鍵：閘只能引用固定偏移的列（cur / next / prev），所以演算法要排成規則的縱向流水線 —— 這就是我們把 Horner 寫成「逐列遞推」的原因。一般地，一個閘是 \(\textstyle\sum_i s_i\,g_i(\text{cells})=0\)，每個 \(g_i\) 是多元多項式、每列須為 0。
PART II · 第 2 層（續）

把 cell 接起來：複製約束 與 查表

🎯 真正要驗的就這兩條：複製約束看 \(Z_n=1\)（繞一圈乘積回到 1 ⟺ 該相等的 cell 都相等）；查表看 \(\sum_i\frac{1}{X+f_i}=\sum_j\frac{m_j}{X+t_j}\)（成立 ⟺ 每個查值都在表內）。
原文 · Equality (copy) constraints “A sequence of equality constraints, which specify that two given cells must have equal values.” The halo2 Book ↗
作法：排列大乘積（PLONK permutation）
\[ Z_{i+1}=Z_i\cdot\frac{a_i+\beta\,\mathrm{id}(i)+\gamma}{a_i+\beta\,\sigma(i)+\gamma},\quad Z_0=Z_n=1 \]
\(Z_n=1\) ⟺ \(\{a_i\}\) 在 \(\mathrm{id}\) 與置換 \(\sigma\) 下是同一個多重集 → 被 \(\sigma\) 串在一起的 cell 必相等。\(\beta,\gamma\) 為 Fiat–Shamir 挑戰。
原文 · plookup（原始構想） “We present a protocol for checking the values of a committed polynomial … over a multiplicative subgroup … are contained in the values of a table.” Gabizon & Williamson — ePrint 2020/315 ↗
作法：logUp 對數導數（Haböck 2022/1530）
\[ \sum_{i}\frac{1}{X+f_i}=\sum_{j}\frac{m_j}{X+t_j} \]
\(m_j\)=重數。表 \(t_j=\{0,\dots,255\}\)（ByteTable）做逐 byte 範圍檢查。機制詳見 gates.html
PART II · 形式推導（聽不懂可跳過，不影響主線）

算術化 → 多項式 → 整除 Z_H

🎯 真正要驗的就這條：\(G(X)=t(X)\cdot Z_H(X)\) —— 「每一列都滿足約束」\(\Longleftrightarrow Z_H\mid G\)。下面 ①–④ 都是在推出這個等價。
  1. 求值域：取 \(n=2^k\) 階乘法子群 \(H=\langle\omega\rangle=\{1,\omega,\dots,\omega^{\,n-1}\}\)；vanishing 多項式 \(Z_H(X)=X^{n}-1\)（在 \(H\) 上恆為 0，因為 \((\omega^i)^n=1\)）。
  2. 每欄插值成多項式：令 advice \(a(\omega^{\,i})=\) 第 \(i\) 列的值（Lagrange 插值）；fixed / selector 同理記為 \(q(X)\)。rotation「下一列」\(=\) 代入 \(\omega X\)（因 \(a(\omega\cdot\omega^i)=a(\omega^{i+1})\)）。
  3. 所有約束合成一條（每條約束各乘 \(\alpha\) 的一個次方、疊起來；含閘、複製約束、查表）：\[ G(X)=g_0(X)+\alpha\,g_1(X)+\alpha^{2}g_2(X)+\dots=\sum_{j}\alpha^{\,j}\,g_j(X) \]
  4. 核心等價：每一列都滿足約束 \(\ \Longleftrightarrow\ G\) 在 \(H\) 上恆為 0 \(\ \Longleftrightarrow\ Z_H\mid G\ \Longleftrightarrow\ \exists\,t(X):\) \[ G(X)=t(X)\cdot Z_H(X) \]

PART II · 第 3 層

多項式承諾 KZG：把一條多項式壓成一個群元素

💡 一句話核心：把一整條多項式壓成「一個點」（一個群元素）。怎麼壓？把 f 的係數乘上 SRS 的對應點、全部相加 —— 這一個點就代表整條 f。
原文 · Polynomial commitment scheme “A polynomial commitment scheme is a triplet S = (gen, com, open) such that gen(d) … outputs a structured reference string (SRS) srs; com(f, srs) … returns a commitment cm to f; open is a public coin protocol between [prover] and [verifier].” gen＝產 SRS｜com＝把 f 壓成承諾 cm（一個群元素）｜open＝證某點的值。本頁專講 com；open ＋ 驗證 → 下一張。 Boneh, Drake, Fisch, Gabizon — ePrint 2020/081, Def. 2.3 ↗
com 怎麼算（Kate–Zaverucha–Goldberg, 2010） 把 f 的係數乘 SRS 對應點再相加：\[ \mathrm{com}(f)=[f(\tau)]_1=\textstyle\sum_i f_i[\tau^i]_1=f_0[1]_1+f_1[\tau]_1+f_2[\tau^2]_1+\cdots \]SRS \(=\{[\tau^i]_1\},[\tau]_2\)，通用可更新、\(\tau\) 用完即丟。 Kate, Zaverucha, Goldberg — ASIACRYPT 2010 ↗

關鍵：prover 不知道 \(\tau\)，只是把公開的 SRS 點做線性組合 —— 從結果反推不出 \(\tau\) 或 \(f\)（離散對數難）。套例子：承諾 \(a(X)=[2,9,31,98]\) 插值 → \(C=[a(\tau)]_1\)。

PART II · 形式推導（聽不懂可跳過，不影響主線）

KZG：逐步形式推導

🎯 真正要驗的就這條：\(e\big(C-[v]_1,[1]_2\big)=e\big(\pi,[\tau]_2-[z]_2\big)\) —— 在秘密點 \(\tau\) 確認 \(f(z)=v\)。下面 ①–⑦ 都是在湊出它、或證明它夠。
  1. 記號 & SRS：\([x]_1:=x\,g_1,\ [x]_2:=x\,g_2\)；\(\ \mathrm{srs}=\big([\tau^i]_1\big)_{i=0}^{d},\,[1]_2,[\tau]_2\)。秘密 \(\tau\) 用完即銷毀。（加法記號，即乘法群的 \(g_1^{\,x}\)）
  2. 承諾（用 srs，不需知道 \(\tau\)）：\[ C=[f(\tau)]_1=\sum_{i=0}^{d} f_i\,[\tau^i]_1 \]
  3. 關鍵恆等式：\[ f(z)=v\ \Longleftrightarrow\ (X-z)\mid\big(f(X)-v\big)\ \Longleftrightarrow\ \exists\,q(X):\ f(X)-v=q(X)\,(X-z) \]
  4. 開啟：\[ q(X)=\frac{f(X)-v}{X-z},\qquad \pi=[q(\tau)]_1 \]
  5. 驗證（一次配對檢查）：\[ e\big(C-[v]_1,\ [1]_2\big)\ \overset{?}{=}\ e\big(\pi,\ [\tau]_2-[z]_2\big) \]
  6. 為何成立（雙線性 \(e([a]_1,[b]_2)=e(g_1,g_2)^{ab}\)）：左 \(=e(g_1,g_2)^{f(\tau)-v}\)，右 \(=e(g_1,g_2)^{q(\tau)(\tau-z)}\)；相等 \(\Leftrightarrow f(\tau)-v=q(\tau)(\tau-z)\)，即恆等式在 \(X=\tau\) 成立。
  7. 可靠性：\(f(z)\neq v\Rightarrow(X-z)\nmid(f-v)\)，無合法 \(q\)，偽造機率 \(\le\mathrm{negl}\)（AGM/q-DLOG，\(\tau\) 為秘密隨機點）。

大小：\(|C|=|\pi|=1\) 個 \(G_1\)、2 配對，與 \(\deg f\) 無關。具體開啟見下一張範例。

PART II · 第 3 層 · 範例

KZG 範例：開啟一個值（quotient 怎麼來）

🎯 真正要驗的就這條：\((X-z)\mid(f-v)\)（這裡 \(X^3-27\) 被 \(X-3\) 整除）⟺ \(f(3)=27\)，驗證落到一條配對 \(e(C-[v]_1,[1]_2)=e(\pi,[\tau]_2-[z]_2)\)。下面用具體數字走一遍。

要證「多項式 \(f\) 在 \(z\) 的值是 \(v\)」。用乾淨小例：取 \(f(X)=X^3,\ z=3,\ v=27\)。

  1. 靈魂恆等式：\(f(z)=v\ \Longleftrightarrow\ (X-z)\mid\big(f(X)-v\big)\)。這裡 \(f-v=X^3-27\)。為什麼這個等價成立？ 餘式定理：\(f(X)-v\) 除以 \((X-z)\) 的餘數恰好 \(=f(z)-v\)（一個常數）。所以 餘數 \(=0\ \Leftrightarrow\ f(z)=v\ \Leftrightarrow\) 整除 —— 「能不能整除」本身就是一台測謊機。
  2. 算商多項式 \(q\)（= 開啟證明的核心）：\[ q(X)=\frac{X^3-27}{X-3}=X^2+3X+9 \]驗算：\((X^2+3X+9)(X-3)=X^3-27\ ✓\)。能整除 ⟺ \(f(3)=27\) 真的成立。
  3. 承諾 + 開啟：\(C=[f(\tau)]_1\)；開啟證明 \(\pi=[q(\tau)]_1=[\tau^2+3\tau+9]_1\)，單一群元素。
  4. 驗證（一條配對）：\(e\big(C-[v]_1,[1]_2\big)\overset{?}{=}e\big(\pi,[\tau]_2-[z]_2\big)\)；雙線性下兩邊都 \(=e(g_1,g_2)^{\,f(\tau)-27}\)，相等就過。
重點：為什麼這證明了 f(3)=27
prover 算得出 \(q\)（整除成功）就證明了 \(f(3)=27\)；他若說謊（值不對），\((X-3)\) 不整除 \(f-v\)，根本沒有合法的 \(q\)，偽造機率可忽略。\(C,\pi\) 是 254-bit 群元素，真實位元組在 Part V 跑出來。
PART II · 第 3 層（續）· 金鑰怎麼來

Setup → Keygen：pk / vk 裡到底有什麼

💡 一句話核心：pk 包含 vk；換電路只重跑 keygen、不必重做 setup —— 通用 SRS 勝過 Groth16「每電路一次設定」。

「承諾誰」？電路裡固定不變的部分（閘係數、selector、wiring）要先預處理、先承諾好 —— 這步叫 keygen，分兩段：

① setup —— 與電路無關、一次性
產生通用 SRS \(=\{[\tau^i]_1\}_{i=0}^{d},[\tau]_2\)（KZG 的 powers-of-τ）。可信設定、可更新、\(\tau\) 用完銷毀。一份，所有電路共用。ParamsKZG::setup(k)
② keygen —— 與這個電路綁定
拿 SRS + 電路，把固定部分用 KZG 承諾起來 → 產出 \((\mathrm{vk},\mathrm{pk})\)。keygen_vk / keygen_pk

vk 驗證金鑰 小 · 公開 · 可上鏈

  • evaluation domain（\(2^k\) 大小）
  • fixed / selector 欄的承諾（KZG）
  • 置換論證的承諾（wiring \(\sigma\)）
  • constraint system（閘 / 查表結構）
  • vk 的 transcript hash（綁進 Fiat–Shamir）

= 驗證者需要的最小集合。

pk 證明金鑰 大 · prover 私用

  • 整個 vk（pk ⊇ vk）
  • fixed / selector：係數式 + Lagrange 求值 + 延伸 coset
  • 置換多項式 \(\sigma\)（多種形式）
  • Lagrange 基 \(l_0,l_{\text{last}},l_{\text{blind}}\)（邊界 / 盲化）

= 用來真正算商多項式、產生證明的全部預處理料。

setup / keygen 是一次性前置（不在 ①–⑥ 流程裡），產物從承諾步驟起才用到。對照 Part V demo：Stage 1 = setup、Stage 2 = keygen（halo2_proofs keygen_vk / keygen_pk）。
PART II · 第 4 層

批次開啟 SHPLONK：為什麼證明又小又不隨深度長

💡 一句話核心：把幾十個開啟批次成 2 個群元素、2 次配對 —— 不管電路多深，證明大小幾乎不變。
原文 · SHPLONK “We present an enhanced version of the Kate, Zaverucha and Goldberg polynomial commitment scheme where a single group element can be an opening proof for multiple polynomials each evaluated at a different arbitrary subset of points.” 把「多條多項式、各在不同點開啟」批次成單一開啟證明（其方案 2：2 個 G₁ 元素），驗證僅 2 次配對 → 證明小、驗證快。 Boneh, Drake, Fisch, Gabizon — ePrint 2020/081 ↗
為什麼要批次？（KZG 的 open 不是夠了嗎）
KZG 的 open = 一條多項式、一個點 → 一個 \(\pi\)。但整條協議要開啟一大堆：每條 advice、\(Z\)、商 \(t\) 的每塊、fixed… 各在 \(\zeta\)（有的還要 \(\omega\zeta\)），動輒 10~30 個。逐個單開 → \(N\) 個群元素 + \(N\) 次配對，隨 \(N\) 線性長，KZG 常數大小的優勢就沒了。

Fiat–Shamir：把互動挑戰換成對 transcript 的雜湊；PSE 預設 Blake2b → 非互動。

🔗 接續例子：\(a(X)\)、\(G(X)\)、\(t(X)\)、\(Z(X)\) 在 \(\zeta\)／\(\omega\zeta\) 的開啟，批次成一個 \(\pi\)。

PART II · 形式推導（聽不懂可跳過，不影響主線）

SHPLONK：怎麼做到批次開啟

🎯 真正要驗的就這條：構造一條 \(L\) 使 \(L(z)=0\) —— 把「多條多項式、多個點」全壓成一次 KZG 開啟。下面 ①–④ 都在湊出這個 \(L\)，最終證明 \(=(W,W')\) 只有 2 個 \(G_1\)。
一招打天下：用一個隨機數，把「很多條」壓成「一條」
暖身（先看同一點）：要證 \(f_1,\dots,f_k\) 在同一點 \(z\) 的值，不必各開一次。verifier 給隨機 \(\gamma\)，prover 改證一條組合 \(F=\sum_i\gamma^{\,i-1}f_i\) 在 \(z\) 的值 —— 任一條造假，隨機組合幾乎必錯，所以一次開啟就驗全部。這就是批次的靈魂。SHPLONK 只是把這招推廣到「各條開在不同點」。
  1. 歸一化（讓不同點也能一起處理）：各 \(f_i\) 在點集 \(S_i\) 開啟。減掉你聲稱的值 \(r_i\)（\(f_i\) 在 \(S_i\) 的插值）、除掉它該為零的 \(Z_{S_i}=\prod_{s\in S_i}(X-s)\) → 得乾淨的商 \((f_i-r_i)/Z_{S_i}\)。
  2. 用 \(\gamma\) 折「多條」→ 一條：把這些商線性疊成 \(h(X)=\sum_i\gamma^{\,i-1}(f_i-r_i)/Z_{S_i}\)，承諾一個 \(W=[h]_1\)。
  3. 用 \(z\) 折「多點」→ 一點：再構造一條 \(L\) 使 \(L(z)=0\) —— 等於把「多點檢查」退化回我們已會的單次 KZG 開啟，再承諾一個 \(W'=[L/(X-z)]_1\)。（\(L\) 的明確式見講稿）
  4. 驗證：verifier 靠承諾的同態自己組出 \([L]_1\)，只做一條配對檢查。
結果：證明 \(=(W,W')\) 只有 2 個 \(G_1\) 元素、2 次配對，與「開幾個」\(N\) 無關。
這是誰的技術？SHPLONK 屬於承諾層（KZG），不是 Halo2 算術化的一部分，也非 Halo2 獨有 —— BDFG ePrint 2020/081 提出、PLONK 家族通用；原版 IPA 的 Halo2 不用 SHPLONK（IPA 自己折在內積論證裡）。呼應〈兩層架構〉：換承諾層，就換批次法。
PART II · 形式推導（聽不懂可跳過，不影響主線）

協議 → 在隨機點 ζ 驗一條等式

🎯 真正要驗的就這條：\(G(\zeta)=t(\zeta)\,(\zeta^{n}-1)\) —— 只在一個隨機點 \(\zeta\) 抽查整除性（Schwartz–Zippel：隨機點成立 ⟹ 處處成立）。下面 ①–⑤ 是湊出它的回合。

Fiat–Shamir 的順序是 soundness 關鍵：每個挑戰都必須在「相關承諾送出後」才抽，所以是承諾 ↔ 挑戰交錯的回合，不是一次全送、一次全抽。

  1. Round 1 — 承諾 advice：commit 各 advice 欄 \(a(X),\dots\)（先加 blinding 隨機項 → 零知識）。
  2. FS → \(\beta,\gamma\) ；Round 2 — 承諾 grand-product：用 \(\beta,\gamma\) 建排列 / 查表的 \(Z(X)\) 並承諾。
  3. FS → \(\alpha\) ；Round 3 — 算商並承諾：\(t(X)=G(X)/Z_H(X)\)；因 \(t\) 次數高，切成多塊 \(t_0,t_1,\dots\)（每塊次數低於 \(n\)）各自承諾。
  4. FS → \(\zeta\) ；開啟：在 \(\zeta\)（rotation 用 \(\omega\zeta\)）開啟所有多項式 → \(a(\zeta),Z(\zeta),Z(\omega\zeta),t(\zeta),\dots\)，附 KZG 開啟證明。
  5. 驗證 + 為何夠：代入檢查 \(G(\zeta)\overset{?}{=}t(\zeta)\,(\zeta^{n}-1)\)（接續例子 \(n=4\)：驗 \(G(\zeta)=t(\zeta)(\zeta^{4}-1)\)）；Schwartz–Zippel → 在隨機 \(\zeta\) 成立 \(\Rightarrow\) 在 \(H\) 上處處成立。

所有開啟 → SHPLONK 批次。整條 Halo2 = 算術化 → 整除 Z_H → 回合式承諾↔挑戰 → ζ 一條等式 + 配對。

PART III · 邊界

Halo2 能表達什麼 · 不能表達什麼

PART IV · 詳細作法（公式 · 聽不懂可跳過，不影響主線）

定點 Horner kernel — 逐步公式

🎯 真正要驗的就這條：\(\mathrm{acc}_k\!\cdot\! R+c_k=\mathrm{acc}_{k+1}\!\cdot\! S+\mathrm{rem}_k,\ \mathrm{rem}_k\in[0,S)\) —— 餘數落在範圍內 ⟺ 誠實做了 \(\div S\) 截斷。下面 ①–⑤ 把它拆成可查表的閘。
  1. 定點表示：取尺度 \(S = 2^{pp},\ pp=48\)。實數 \(v\) 編碼成整數 \(\hat v = \lfloor v\cdot S\rceil\)（所有輸入都先這樣進電路）。
  2. 多項式逼近：用 Remez minimax 求係數，\( \; P(x)=\sum_{i=0}^{d} c_i\,x^{i}\; \)（係數定點化後存進 fixed 欄）。
  3. 逐步截斷 Horner（輸入 \(R=\hat x\)，初值 \(\mathrm{acc}_0=c_d\)）：\[ \mathrm{acc}_{k+1}=\Big\lfloor \tfrac{\mathrm{acc}_k\cdot R + c_k}{S}\Big\rfloor,\quad k=0,\dots,d-1,\qquad \mathrm{acc}_d=\widehat{P(x)} \]
  4. 閘 — 強制誠實截斷（\(\mathrm{rem}_k=\mathrm{acc}_kR+c_k-\mathrm{acc}_{k+1}S\)）：\[ q\cdot\big(\mathrm{acc}_k\!\cdot\! R + c_k-\mathrm{acc}_{k+1}\!\cdot\! S-\mathrm{rem}_k\big)=0,\quad \mathrm{rem}_k\in[0,S) \]
  5. 範圍檢查 — lookup（\(S=2^{48}=256^{6}\)）：\[ q\cdot\Big(\mathrm{rem}_k-\sum_{j=0}^{5} b_{k,j}\,256^{\,j}\Big)=0,\quad b_{k,j}\in\{0,\dots,255\}\ \text{（查 ByteTable）} \]

第 4、5 條就是 demo 那條閘 \(q\,(\mathrm{acc}\cdot x+c-\mathrm{acc}')\) 多出來的部分 —— \(\div S\) 截斷 + 餘數 byte 拆解。換一個函數只換 \(\{c_i\}\)，電路不動。

fn configure(meta: &mut ConstraintSystem<Fr>) -> HornerConfig {
    let acc = meta.advice_column();   // 私密：累加器
    let x   = meta.advice_column();   // 私密：求值點
    let coeff = meta.fixed_column();  // 常數：每步係數
    let instance = meta.instance_column(); // 公開：P(x)
    let s = meta.selector();
    meta.enable_equality(acc); meta.enable_equality(x); meta.enable_equality(instance);
    meta.create_gate("horner step", |meta| {        // 整個電路就這一條恆等式
        let s = meta.query_selector(s);
        let acc_cur = meta.query_advice(acc, Rotation::cur());
        let x_cur   = meta.query_advice(x,   Rotation::cur());
        let c_cur   = meta.query_fixed (coeff, Rotation::cur());
        let acc_nxt = meta.query_advice(acc, Rotation::next());
        vec![ s * (acc_cur * x_cur + c_cur - acc_nxt) ]   // = 0
    });
    HornerConfig { acc, x, coeff, s, instance }
}

附錄 · 現代 ZK 在哪裡用

現代 ZK 已經大規模落地 —— 我們的論文就在這個 landscape

一句話收：經典 ZK 為單一 statement 設計協議；現代 ＝ 算術化 + 多項式承諾（PLONK 範式）。重點是它已經到處在用：

zkEVM / zkRollup

以太坊擴容 —— Scroll、Taiko、zkSync 把大量交易鏈下證明、鏈上只驗一個。

zkML

證明「這個機器學習推論真的跑對」，不洩漏模型 / 輸入。

償付能力（proof-of-solvency）

交易所證明「準備金足額」，不公開個別帳戶。

合規證明 ← 我們的 PF1

投組風險限額：證明 \(|\sum_i w_i\Delta_i|\le\tau\)，不洩漏個別部位。我們的論文就坐在這格。

仍有的硬限制：沒有原生浮點（正是我們論文在解的）、非後量子。

收尾

如果只記得五件事

1

Halo2 = PLONKish 算術化 + 多項式承諾；我們用 PSE 的 KZG/BN254 版（上鏈效率）。

2

電路 = 一張 2ᵏ 列的表 + 一組「恆等於 0」的多項式閘，cell 用複製約束與查表接線。

3

KZG + SHPLONK 讓證明又小（demo 864 B）又快驗（~2 ms），且大小不隨深度成長。

4

Halo2 沒有浮點 → 我們用定點 + Remez + 逐步 Horner 閘重建超越函數，這就是論文貢獻。

5

同一條閘、同一套 pipeline 從 demo 一路長到 PF1/PF2 真實電路 — 已實作、已驗證。
