Halo2 技術專題2026 · 畢業論文 v4
① 在做什麼

Halo2 = 一台「可驗證計算」的引擎

證明者向驗證者證明「我正確執行了某段計算」，而不洩漏私密輸入，且證明又小又快驗。

zk-SNARK. Zero-Knowledge Succinct Non-interactive ARgument of Knowledge — 簡潔（證明遠小於計算）、非互動、零知識的知識論證。

家譜

  • Halo（Bowe·Grigg·Hopwood, 2019，ePrint 2019/1021）：首個免信任設定的遞迴證明組合。
  • Halo2（Electric Coin Company / Zcash）：把 Halo 的想法重建在 PLONKish 算術化之上，成為通用電路框架。

一句話定位

Halo2 不是「一個」協議，而是「算術化層 + 多項式承諾層」的組合框架 —
你描述電路，它負責產生 / 驗證證明。

下一頁就是這兩層，以及我們選了哪個版本。

在做什麼03
① 在做什麼 · 關鍵分支

兩層架構，與一個必須講清楚的分叉

上層：PLONKish 算術化

把計算排成一張表格（trace table）＋一組多項式恆等式。
這一層與下層的承諾方案無關 — 兩個版本共用。

下層：多項式承諾方案 (PCS)

把表格編碼成多項式並「承諾」，用代數方式檢查恆等式是否成立。

版本承諾曲線信任設定典型用途
原版 ECC / ZcashIPA（內積論證）Pasta（Pallas/Vesta）免（遞迴友善）Zcash Orchard
PSE fork ← 我們用這個KZGBN254需通用 SRSEVM 上鏈驗證

為什麼選 PSE / KZG / BN254？

BN254 的配對是以太坊內建 precompile → 鏈上驗證便宜；KZG 證明大小固定、驗證只需常數次配對。代價是需要一次性的通用可信設定（universal SRS）。

在做什麼04
② 怎麼做 · 第 1 層

算術化：把程式攤成一張 2ᵏ 列的表

Trace table. 一個 2ᵏ × w 的矩陣 over 𝔽ᵣ（BN254 純量體，r ≈ 2²⁵⁴）。每一欄是一條多項式，每一列是計算的一步。

四種欄

  • Advice 私密見證（prover 的秘密）
  • Fixed 電路常數（如多項式係數）
  • Instance 公開輸入 / 輸出
  • Selector 0/1 開關，決定某列哪個閘生效

範例：一張 4 列的表

 row │ acc(adv) │ x(adv) │ coeff(fix) │ sel
─────┼──────────┼────────┼────────────┼────
  0  │    2     │   3    │     3      │  1
  1  │    9     │   3    │     4      │  1
  2  │   31     │   3    │     5      │  1
  3  │   98     │   ·    │     ·      │  0
                              ↑ instance = 98

這正是稍後 demo 的表 — 用 Horner 算 P(x)。

怎麼做05
② 怎麼做 · 第 2 層

自訂閘：每條規則 = 一個 恆等於 0 的多項式

Custom gate. 一個多元多項式 g(...)，在所有 selector=1 的列上必須 g = 0。它能引用同列與相鄰列（rotation）的 cell。

我們論文的核心閘：一步 Horner

// 在 sel=1 的列上強制成立：
sel * ( acc·x + coeff − acc_next ) = 0
       └ acc_next 用 Rotation::next() 讀下一列

係數 d 個的多項式 P(x)=Σcᵢxⁱ，用 d 列就能逐步算完：
acc₀=c_d，accᵢ₊₁ = accᵢ·x + c。

為什麼是「鄰列」很重要

Halo2 的閘只能看固定偏移的列（cur / next / prev / ±k）。所以演算法必須排成規則的縱向流水線 — 這直接決定了我們把 Horner 寫成「逐列遞推」而不是一次展開。

閘的次數上限（本電路 ≤ 6）決定 quotient 多項式要擴張多大 → 影響 prover 成本。

怎麼做06
② 怎麼做 · 第 2 層（續）

把 cell 接起來：複製約束 與 查表

複製約束 (copy constraint)

Permutation argument. 用 PLONK 的排列「大乘積」恆等式，斷言任意兩個 cell 相等。

例：demo 裡私密的 x 在每一 Horner 列都要是同一個值 → 用複製約束把它們綁在一起，惡意 prover 不能每列換一個 x。

查表論證 (lookup)

plookup. Gabizon·Williamson 2020，ePrint 2020/315。斷言一組 cell 的值落在一張預先定義的表裡。

用途：範圍檢查、S-box、位元運算。
我們用它做 ByteTable[0..255] 範圍檢查 — 保證定點截斷的餘數每個 byte 都合法。

→ 算術化層到此完整：欄 + 自訂閘 + 複製約束 + 查表。接下來把它變成密碼學證明。

怎麼做07
② 怎麼做 · 第 3 層

多項式承諾 KZG：把一條多項式壓成一個群元素

KZG (Kate–Zaverucha–Goldberg, 2010). 對多項式 f 的承諾就是單一群元素 [f(τ)]₁；要「開啟」某點的值，用一次配對檢查驗證。

SRS — 結構化參考字串

  • 通用 (universal)：一次設定，所有 ≤ 2ᵏ 的電路共用。
  • 可更新 (updatable)：只要設定儀式中有一人誠實就安全。
  • 內容是秘密 τ 的冪次 {[τⁱ]₁}, [τ]₂，τ 用完即丟（toxic waste / trapdoor）。

這就是「可信設定」的代價

原版 Halo（IPA）不需要它；PSE/KZG 為了 BN254 上鏈效率接受它，並靠 Perpetual Powers of Tau 公開儀式把信任降到最低。

我們論文如實揭露這是一條信任假設（trust assumption），不宣稱 trustless。

怎麼做08
② 怎麼做 · 第 4 層

批次開啟 SHPLONK + Fiat–Shamir

SHPLONK. Boneh·Drake·Fisch·Gabizon, 2020，ePrint 2020/081。把「在多個點開啟多條多項式」批次成近乎常數大小的單一開啟證明。

結果：證明大小 ≈ O(欄數)，不隨列數（計算深度）成長。驗證只需常數次配對 + 一個 MSM。

Fiat–Shamir. 把互動式挑戰換成對 transcript 的雜湊。PSE v0.3.0 預設用 Blake2b（Blake2bWrite/Read）→ 非互動。

為什麼證明這麼小？

關鍵：承諾把「整條多項式」壓成一個點，SHPLONK 再把「許多開啟」併成一個。所以列數翻倍、proof 幾乎不變。

③ 為什麼能用

為什麼「檢一個隨機點」就足以信任整條多項式？

Schwartz–Zippel 引理. 兩個次數 ≤ d 的相異多項式，在 |𝔽| ≈ 2²⁵⁴ 的體上隨機取一點相等的機率 ≤ d / |𝔽| → 可忽略。所以「在 Fiat–Shamir 抽的隨機點上恆等式成立」⇒ 幾乎必然處處成立。
性質意義機制
完備性誠實 prover 必過合法 witness 滿足所有閘
可靠性作弊者過不了承諾綁定 + Schwartz–Zippel + 抽取器
零知識不洩漏 adviceadvice 多項式加隨機 blinding
簡潔性又小又快驗KZG + SHPLONK 常數配對

安全立基

KZG 的知識可靠性建立在配對友善曲線的代數假設（AGM + q-type / ROM）。BN254 提供約 128-bit 安全等級。

→ 我們論文 ch3 對這三項（completeness / soundness / zero-knowledge）給了形式化論證。

Halo2 能表達什麼 · 不能表達什麼

✅ 直接能做

  • 𝔽ᵣ 上的加 / 乘（任意算術）

  • 複製約束 / 任意 wiring

⚠️ 需 gadget 或有硬限制

  • k ≤ ~28（k=28 約 30 GB RAM）
  • 位元運算需 boolean 拆解（SHA/Keccak 貴 2 個數量級）

  • 浮點無原生支援 → 我們用 pp=48 定點
  • 資料相依的控制流不允許（trace 預先固定）

「浮點不支援」這一條，正是整篇論文存在的理由 → 下一節。

⑥ 舉例 · 真實程式碼 (1/3)

Demo 電路：欄位 + 那一個閘

敘述：「我知道一個秘密 x，使 P(x)=2x³+3x²+4x+5 等於某公開值。」x=3 → P(3)=98。

fn configure(meta: &mut ConstraintSystem<Fr>) -> HornerConfig {
    let acc   = meta.advice_column();   // 私密：Horner 累加器
    let x     = meta.advice_column();   // 私密：求值點
    let coeff = meta.fixed_column();    // 常數：每步係數
    let instance = meta.instance_column(); // 公開：P(x)
    let s = meta.selector();

    meta.enable_equality(acc); meta.enable_equality(x); meta.enable_equality(instance);

    // 整個電路就是這一條多項式恆等式：
    meta.create_gate("horner step", |meta| {
        let s       = meta.query_selector(s);
        let acc_cur = meta.query_advice(acc,   Rotation::cur());
        let x_cur   = meta.query_advice(x,     Rotation::cur());
        let c_cur   = meta.query_fixed (coeff, Rotation::cur());
        let acc_nxt = meta.query_advice(acc,   Rotation::next());
        vec![ s * (acc_cur * x_cur + c_cur - acc_nxt) ]   // = 0
    });
    HornerConfig { acc, x, coeff, s, instance }
}

let mut acc = region.assign_advice(
    || "acc0=c3", acc, 0, || known(c3))?;
let x0 = region.assign_advice(|| "x", x, 0, || self.x)?;
for i in 0..3 {
    s.enable(&mut region, i)?;

    acc = region.assign_advice(|| "acc", acc, i+1, || nxt)?;
}

1
