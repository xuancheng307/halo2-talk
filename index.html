PART I · 在做什麼

Halo2 的家譜

原文 · Halo “We obtain and implement Halo, the first practical example of recursive proof composition without a trusted setup, using the discrete log assumption over normal cycles of elliptic curves.” — 首個「免信任設定」的實用遞迴證明組合，建立在橢圓曲線離散對數假設上。 Bowe, Grigg, Hopwood — IACR ePrint 2019/1021 ↗

一句話定位

Halo2 不是「一個」協議，而是 「算術化層 + 多項式承諾層」 的組合框架 — 你描述電路，它負責產生／驗證證明。

下一節：它跟既有技術（Bulletproofs / Groth16）差在哪。

PART I · 為什麼要有 Halo2

與既有技術的差異：Bulletproofs · Groth16 · Halo2

兩條既有路線 — Bulletproofs：把證明 reduce 到離散對數（IPA）；Groth16：pairing + 每電路一次可信設定。Halo2 走第三條：算術化 + 多項式承諾。

PART I · 原文背書（不是我說的）

三者的安全「定義」相同，差別在建構

原文 · Bulletproofs “We propose Bulletproofs, a new non-interactive zero-knowledge proof protocol with very short proofs and without a trusted setup … relying on the discrete logarithm assumption … Proof generation and verification times are linear in n.” Bünz, Bootle, Boneh, Poelstra, Wuille, Maxwell — ePrint 2017/1066 ↗
原文 · Groth16 “… a proof is only 3 group elements, and verification consists of checking a single pairing product equation using 3 pairings in total.” Jens Groth — EUROCRYPT 2016, ePrint 2016/260 ↗

那座橋（接回既有技術）

三者的安全性質是同一套（completeness / knowledge soundness / zero-knowledge）—— 差別在建構與信任模型，不在定義。關鍵接點：原版 Halo2 的承諾用 IPA — 就是 Bulletproofs 的離散對數家族，你們認得；我們用的 PSE 版只是把這層換成 KZG（pairing 多項式承諾），為了通用設定與快驗證。

PART I · 現代範式

這條「第三路」的名字叫 PLONK

從 Sigma / Schnorr、Groth16 到現代通用 SNARK 之間的範式轉移，就是 PLONK（2019） —— 而 Halo2 是它的一個實作。

原文 · PLONK “We present a universal SNARK construction with fully succinct verification, and significantly lower prover running time … we focus on ‘Evaluations on a subgroup rather than coefficients of monomials’, which enables simplifying both the permutation argument and the arithmetization step.” Gabizon, Williamson, Ciobotaru — ePrint 2019/953 ↗

PLONK 帶來的三個「新」

  • 通用 + 可更新 SRS：一次設定，任意電路共用（不像 Groth16 每電路一次）。
  • 多項式 IOP：電路 → 多項式約束，用承諾 + 隨機點檢查取代「reduce 到單一困難問題」。
  • 自訂閘 + 查表（UltraPLONK / Plonkish）：表達複雜運算。

Halo2 = PLONKish 算術化 + KZG 承諾。你今天要懂的「新東西」就是這個 PLONK 家族。

PART II · 先講清楚：在證什麼

整條流程在證什麼？ — 一個 statement，以及它的限制

① 我們有一個想證的 statement

形式上就是某關係的成員：\((x,w)\in R\)（即 Part I 的 R_C）。命題是：「我知道一個私密 \(w\)，使公開 \(x\) 與 \(w\) 滿足 \(R\)」 —— \(x\) 公開、\(w\) 私密。

demo：「我知道 \(x\) 使 \(P(x)=98\)」；PF1：「我知道一個投組使 \(|\sum_i w_i\Delta_i|\le\tau\)」。

② 這個 statement 的限制（能進 Halo2 的條件）

  • \(R\) 必須能寫成 𝔽ᵣ 上的算術電路（無原生浮點 / 位元）
  • 固定大小：\(2^k\) 列、有界欄；無資料相依分支、迴圈要有上界
  • 約束是低次多項式（degree ≤ 小常數）
  • 本質 = 任何「給 \(w\) 就能 poly-time 驗證」的 NP 關係

③ 接下來：把這個 statement 算術化 —— 編成 \(2^k\) 列 trace table + 約束，就是下一張「完整流程」的第一步。

自訂閘：每條規則 = 一個 恆等於 0 的多項式

原文 · Polynomial constraints “These are multivariate polynomials over F that must evaluate to zero for each row. The variables … may refer to a cell in a given column of the current row, or a given column of another row relative to this one (with wrap-around …).”
原文 · Gate “… a set of constraints controlled by a set of selector columns that are designed to be used together, [we refer to] as a gate.” The halo2 Book ↗
這條閘的多項式形式（demo）
\[ s\cdot\big(\mathrm{acc}\cdot x + c - \mathrm{acc}'\big)=0,\qquad \mathrm{acc}'=\mathrm{acc}\ @\ \texttt{Rotation::next} \]
關鍵：閘只能引用固定偏移的列（cur / next / prev），所以演算法要排成規則的縱向流水線 —— 這就是我們把 Horner 寫成「逐列遞推」的原因。一般地，一個閘是 \(\textstyle\sum_i s_i\,g_i(\text{cells})=0\)，每個 \(g_i\) 是多元多項式、每列須為 0。
PART II · 第 2 層（續）

把 cell 接起來：複製約束 與 查表

原文 · Equality (copy) constraints “A sequence of equality constraints, which specify that two given cells must have equal values.” The halo2 Book ↗
作法：排列大乘積（PLONK permutation）
\[ Z_{i+1}=Z_i\cdot\frac{a_i+\beta\,\mathrm{id}(i)+\gamma}{a_i+\beta\,\sigma(i)+\gamma},\quad Z_0=Z_n=1 \]
\(Z_n=1\) ⟺ \(\{a_i\}\) 在 \(\mathrm{id}\) 與置換 \(\sigma\) 下是同一個多重集 → 被 \(\sigma\) 串在一起的 cell 必相等。\(\beta,\gamma\) 為 Fiat–Shamir 挑戰。
原文 · plookup “We present a protocol for checking the values of a committed polynomial … over a multiplicative subgroup … are contained in the values of a table.” Gabizon & Williamson — ePrint 2020/315 ↗
作法：對數導數（log-derivative / logUp）
\[ \sum_{i}\frac{1}{X+f_i}=\sum_{j}\frac{m_j}{X+t_j} \]
成立 ⟺ 每個查值 \(f_i\) 都落在表 \(\{t_j\}\) 內（\(m_j\)=重數）。我們的表 \(t_j=\{0,\dots,255\}\)（ByteTable），做定點截斷餘數的逐 byte 範圍檢查。
PART II · 形式推導

算術化 → 多項式 → 整除 Z_H

  1. 求值域：取 \(n=2^k\) 階乘法子群 \(H=\langle\omega\rangle=\{1,\omega,\dots,\omega^{\,n-1}\}\)；vanishing 多項式 \(Z_H(X)=X^{n}-1\)（在 \(H\) 上恆為 0，因為 \((\omega^i)^n=1\)）。
  2. 每欄插值成多項式：令 advice \(a(\omega^{\,i})=\) 第 \(i\) 列的值（Lagrange 插值）；fixed / selector 同理記為 \(q(X)\)。rotation「下一列」\(=\) 代入 \(\omega X\)（因 \(a(\omega\cdot\omega^i)=a(\omega^{i+1})\)）。
  3. 所有約束合成一條（用挑戰 \(\alpha\) 的次方；含閘、複製約束、查表）：\[ G(X)=\sum_{j}\alpha^{\,j}\,g_j\big(a(X),\,a(\omega X),\,q(X),\dots\big) \]
  4. 核心等價：每一列都滿足約束 \(\ \Longleftrightarrow\ G\) 在 \(H\) 上恆為 0 \(\ \Longleftrightarrow\ Z_H\mid G\ \Longleftrightarrow\ \exists\,t(X):\) \[ G(X)=t(X)\cdot Z_H(X) \]
具體：欄 → 多項式、閘 → 多項式（demo, n=4）
acc 欄 \([2,9,31,98]\) 插值成 \(a(X)\)（\(a(\omega^i)=\)該列值）；x→\(x(X)\)、coeff→\(c(X)\)、sel→\(s(X)\)。閘整條換成一條多項式 \(g(X)=s(X)\big(a(X)x(X)+c(X)-a(\omega X)\big)\)（「下一列」\(=a(\omega X)\)）。「閘逐列成立」\(\Leftrightarrow g(\omega^i)=0\ \forall i \Leftrightarrow Z_H\mid g\)。多條約束用隨機 \(\alpha\) 疊成一條 \(G\)。
PART II · 第 3 層

多項式承諾 KZG：把一條多項式壓成一個群元素

原文 · Polynomial commitment scheme “A polynomial commitment scheme is a triplet S = (gen, com, open) such that gen(d) … outputs a structured reference string (SRS) srs; com(f, srs) … returns a commitment cm to f; open is a public coin protocol between [prover] and [verifier].” Boneh, Drake, Fisch, Gabizon — ePrint 2020/081, Def. 2.3 ↗
KZG 的具體做法 “Constant-Size Commitments to Polynomials and Their Applications.” com(f) = [f(τ)]₁ 單一 G₁ 群元素；某點開啟用一次配對檢查。SRS = {[τⁱ]₁}, [τ]₂，通用、可更新（一人誠實即安全）、τ 用完即丟。 Kate, Zaverucha, Goldberg — ASIACRYPT 2010 ↗
KZG 三步公式（\([x]_1:=x\cdot g_1\)）
\[ \underbrace{C=[f(\tau)]_1}_{\text{承諾}}\qquad \underbrace{q(X)=\tfrac{f(X)-v}{X-z},\ \ \pi=[q(\tau)]_1}_{\text{開啟 } f(z)=v} \] \[ \underbrace{e\big(C-[v]_1,\,[1]_2\big)=e\big(\pi,\,[\tau]_2-[z]_2\big)}_{\text{驗證：一次配對檢查}} \]
PART II · 形式推導

KZG：逐步形式推導

  1. 記號 & SRS：\([x]_1:=x\,g_1,\ [x]_2:=x\,g_2\)；\(\ \mathrm{srs}=\big([\tau^i]_1\big)_{i=0}^{d},\,[1]_2,[\tau]_2\)。秘密 \(\tau\) 用完即銷毀。
  2. 承諾（用 srs，不需知道 \(\tau\)）：\[ C=[f(\tau)]_1=\sum_{i=0}^{d} f_i\,[\tau^i]_1 \]
  3. 關鍵恆等式：\[ f(z)=v\ \Longleftrightarrow\ (X-z)\mid\big(f(X)-v\big)\ \Longleftrightarrow\ \exists\,q(X):\ f(X)-v=q(X)\,(X-z) \]
  4. 開啟：\[ q(X)=\frac{f(X)-v}{X-z},\qquad \pi=[q(\tau)]_1 \]
  5. 驗證（一次配對檢查）：\[ e\big(C-[v]_1,\ [1]_2\big)\ \overset{?}{=}\ e\big(\pi,\ [\tau]_2-[z]_2\big) \]
  6. 為何成立（雙線性 \(e([a]_1,[b]_2)=e(g_1,g_2)^{ab}\)）：左 \(=e(g_1,g_2)^{f(\tau)-v}\)，右 \(=e(g_1,g_2)^{q(\tau)(\tau-z)}\)；相等 \(\Leftrightarrow f(\tau)-v=q(\tau)(\tau-z)\)，即恆等式在 \(X=\tau\) 成立。
  7. 可靠性：若 \(f(z)\neq v\)，則 \((X-z)\nmid(f(X)-v)\)，無合法 \(q\) → 偽造成功機率 \(\le \mathrm{negl}(\lambda)\)（q-DLOG / Schwartz–Zippel，\(\tau\) 為秘密隨機點）。

大小：\(|C|=|\pi|=1\) 個 \(G_1\) 元素、驗證 = 2 次配對，皆與 \(\deg f\)（電路大小）無關。純 KZG 只綁定；要零知識需在 \(f\) 加 blinding。

PART II · 第 4 層

批次開啟 SHPLONK：為什麼證明又小又不隨深度長

原文 · SHPLONK “We present an enhanced version of the Kate, Zaverucha and Goldberg polynomial commitment scheme where a single group element can be an opening proof for multiple polynomials each evaluated at a different arbitrary subset of points.” 把「多條多項式、各在不同點開啟」批次成單一開啟證明（其方案 2：2 個 G₁ 元素），驗證僅 2 次配對 → 證明小、驗證快。 Boneh, Drake, Fisch, Gabizon — ePrint 2020/081 ↗

Fiat–Shamir：把互動挑戰換成對 transcript 的雜湊；PSE v0.3.0 預設 Blake2b → 非互動。

PART II · 形式推導

協議 → 在隨機點 ζ 驗一條等式

Fiat–Shamir 的順序是 soundness 關鍵：每個挑戰都必須在「相關承諾送出後」才抽，所以是承諾 ↔ 挑戰交錯的回合，不是一次全送、一次全抽。

  1. Round 1 — 承諾 advice：commit 各 advice 欄 \(a(X),\dots\)（先加 blinding 隨機項 → 零知識）。
  2. FS → \(\beta,\gamma\) ；Round 2 — 承諾 grand-product：用 \(\beta,\gamma\) 建排列 / 查表的 \(Z(X)\) 並承諾。
  3. FS → \(\alpha\) ；Round 3 — 算商並承諾：\(t(X)=G(X)/Z_H(X)\)；因 \(t\) 次數高，切成多塊 \(t_0,t_1,\dots\)（每塊次數低於 \(n\)）各自承諾。
  4. FS → \(\zeta\) ；開啟：在 \(\zeta\)（rotation 用 \(\omega\zeta\)）開啟所有多項式 → \(a(\zeta),Z(\zeta),Z(\omega\zeta),t(\zeta),\dots\)，附 KZG 開啟證明。
  5. 驗證 + 為何夠：代入檢查 \(G(\zeta)\overset{?}{=}t(\zeta)\,(\zeta^{n}-1)\)；Schwartz–Zippel → 在隨機 \(\zeta\) 成立 \(\Rightarrow\) 在 \(H\) 上處處成立。

所有開啟 → SHPLONK 批次。整條 Halo2 = 算術化 → 整除 Z_H → 回合式承諾↔挑戰 → ζ 一條等式 + 配對。

PART III · 為什麼能用 / 形式化

四個性質 — 各自怎麼成立（含形式化背書）

可靠性的核心 Schwartz–Zippel：非零、次數 \(d\) 的多項式只在 \(\le d\) 個點為 0 → 隨機 \(r\) 命中機率 \(\Pr[P(r)=0]\le d/|\mathbb F|\)，在 \(|\mathbb F|\approx 2^{254}\) 可忽略。驗一個隨機點 ≈ 處處成立。

性質怎麼成立（機制 + 證明策略）
完備性合法 witness 逐列滿足每條閘 → 恆等式成立 → 必接受

PART III · 邊界

Halo2 能表達什麼 · 不能表達什麼

✅ 直接能做

  • 𝔽ᵣ 上加 / 乘（任意算術）
  • 自訂閘 degree ≤ ~6，含 rotation
  • 查表（固定 / 共享表）
  • 複製約束 / 任意 wiring
  • selector 做 2-路 MUX

⚠️ 需 gadget 或有硬限制

  • k ≤ ~28（k=28 約 30 GB RAM）
  • 位元運算需 boolean 拆解（SHA/Keccak 貴 2 個數量級）
  • 除法 / 模逆：prover 提供 quotient witness
  • 浮點無原生支援 → 我們用 pp=48 定點
  • 資料相依控制流不允許

「浮點不支援」這一條，正是整篇論文存在的理由 → Part IV。

關鍵洞見

PART IV · 詳細作法（公式）

定點 Horner kernel — 逐步公式

  1. 定點表示：取尺度 \(S = 2^{pp},\ pp=48\)。實數 \(v\) 編碼成整數 \(\hat v = \lfloor v\cdot S\rceil\)（所有輸入都先這樣進電路）。
  2. 多項式逼近：用 Remez minimax 求係數，\( \; P(x)=\sum_{i=0}^{d} c_i\,x^{i}\; \)（係數定點化後存進 fixed 欄）。
  3. 逐步截斷 Horner（輸入 \(R=\hat x\)，初值 \(\mathrm{acc}_0=c_d\)）：\[ \mathrm{acc}_{k+1}=\Big\lfloor \tfrac{\mathrm{acc}_k\cdot R + c_k}{S}\Big\rfloor,\quad k=0,\dots,d-1,\qquad \mathrm{acc}_d=\widehat{P(x)} \]
  4. 閘 — 強制誠實截斷（\(\mathrm{rem}_k=\mathrm{acc}_kR+c_k-\mathrm{acc}_{k+1}S\)）：\[ q\cdot\big(\mathrm{acc}_k\!\cdot\! R + c_k-\mathrm{acc}_{k+1}\!\cdot\! S-\mathrm{rem}_k\big)=0,\quad \mathrm{rem}_k\in[0,S) \]
  5. 範圍檢查 — lookup（\(S=2^{48}=256^{6}\)）：\[ q\cdot\Big(\mathrm{rem}_k-\sum_{j=0}^{5} b_{k,j}\,256^{\,j}\Big)=0,\quad b_{k,j}\in\{0,\dots,255\}\ \text{（查 ByteTable）} \]

第 4、5 條就是 demo 那條閘 \(q\,(\mathrm{acc}\cdot x+c-\mathrm{acc}')\) 多出來的部分 —— \(\div S\) 截斷 + 餘數 byte 拆解。換一個函數只換 \(\{c_i\}\)，電路不動。

Demo 電路：欄位 + 那一個閘

敘述：「我知道秘密 x，使 P(x)=2x³+3x²+4x+5 = 某公開值。」x=3 → P(3)=98。

fn configure(meta: &mut ConstraintSystem<Fr>) -> HornerConfig {
    let acc = meta.advice_column();   // 私密：累加器
    let x   = meta.advice_column();   // 私密：求值點
    let coeff = meta.fixed_column();  // 常數：每步係數
    let instance = meta.instance_column(); // 公開：P(x)
    let s = meta.selector();
    meta.enable_equality(acc); meta.enable_equality(x); meta.enable_equality(instance);
    meta.create_gate("horner step", |meta| {        // 整個電路就這一條恆等式
        let s = meta.query_selector(s);
        let acc_cur = meta.query_advice(acc, Rotation::cur());
        let x_cur   = meta.query_advice(x,   Rotation::cur());
        let c_cur   = meta.query_fixed (coeff, Rotation::cur());
        let acc_nxt = meta.query_advice(acc, Rotation::next());
        vec![ s * (acc_cur * x_cur + c_cur - acc_nxt) ]   // = 0
    });
    HornerConfig { acc, x, coeff, s, instance }
}

附錄 · 經典 ZK → 現代 ZK

從經典 ZK 到現代 ZK：更新了什麼

不只 reduce 到離散對數

經典：針對某 statement 設計 Sigma / Schnorr 協議。現代：算術化 + 多項式承諾（PLONK 範式）。

不用每個 statement 設計協議

PLONK 給通用電路框架 — 一套框架描述任意計算，不用為每個問題重新設計。

不用每電路重做 setup

Groth16 每電路一次可信設定；PLONK/KZG 用通用 + 可更新 SRS，一次到處用。

證明不再大又慢

Bulletproofs 驗證線性；KZG 小證明 + 常數驗證（~ms），且不隨計算深度成長。

不只理論 / 隱私貨幣

已大規模部署：zkEVM / zkRollup / zkML、償付能力與合規證明。

仍有的硬限制

沒有原生浮點、非後量子。前者正是我們論文在解的。

收尾

如果只記得五件事

1

Halo2 = PLONKish 算術化 + 多項式承諾；我們用 PSE 的 KZG/BN254 版（上鏈效率）。

2

電路 = 一張 2ᵏ 列的表 + 一組「恆等於 0」的多項式閘，cell 用複製約束與查表接線。

3

KZG + SHPLONK 讓證明又小（demo 864 B）又快驗（~2 ms），且大小不隨深度成長。

4

Halo2 沒有浮點 → 我們用定點 + Remez + 逐步 Horner 閘重建超越函數，這就是論文貢獻。

5

同一條閘、同一套 pipeline 從 demo 一路長到 PF1/PF2 真實電路 — 已實作、已驗證。
